IoT Flood Monitoring System

A dual-unit ESP32 project for real-time water level detection and remote alerting.

Project Overview

This project is a real-world IoT solution designed to address the recurring natural flooding at the Pablo Borbon campus. Due to the area's low-lying geography, certain roads become impassable during heavy rain, posing a risk to drivers. This system provides a clear, automated warning sign to indicate whether the road is safe to traverse.

The system uses a remote Sensor Unit placed at a critical low point. This unit employs two float switches to detect rising water levels. When a water level threshold is breached, it wirelessly signals a central Receiver Unit using the ESP-NOW protocol.

The Receiver Unit controls a highly visible traffic light, giving drivers an immediate and easy-to-understand status of the road ahead. A green light indicates the road is clear, yellow suggests caution as water is rising, and a solid red light warns that the road is impassable. This provides a practical safety measure for a persistent, real-life environmental challenge.

How It Works

State 1: Normal Conditions

When both float switches are dry, the system is in its NORMAL state. The Receiver Unit keeps a standard traffic light cycling (Green -> Yellow -> Red) to regulate normal traffic flow, and the blue flood indicator LED remains off.

State 2: Low-Level Flood

When the lower float switch is triggered by rising water, the system enters the FLOOD_LOW state. The traffic light turns to a 'breathing' or pulsing yellow, signaling to drivers that the road is wet and they should proceed with caution. The blue indicator LED also turns on.

State 3: High-Level Flood

When both the lower and upper float switches are triggered, the system enters the FLOOD_HIGH state, indicating the road is impassable. The traffic light turns to a solid red, and the blue indicator LED stays on, clearly warning drivers not to proceed.

Web Dashboard & Manual Control

The Receiver Unit also hosts a password-protected Wi-Fi Access Point. By connecting to this network, authorized users can view a web dashboard showing the current flood status in real-time. The dashboard also provides manual control over the LEDs, allowing for system resets or tests.

Installation & Setup Guide

1. Software Prerequisites

  • Install the Arduino IDE.
  • Add ESP32 board support to your Arduino IDE. You can find instructions here. Make sure to include support for both standard ESP32 and ESP32-C3 boards.

2. Configure & Flash the Firmware

For the Receiver Unit (ESP32):

  1. First, use the `firmware/MAC_Address_Finder.ino` sketch to get the MAC Address of your ESP32 board. Upload the sketch and copy the MAC address from the Serial Monitor.
  2. Open `firmware/Traffic_Receiver.ino` and select your ESP32 board in the Arduino IDE.
  3. Update the `www_user` and `www_pass` variables if you want to change the web dashboard login credentials.
  4. Upload the `Traffic_Receiver.ino` sketch to your ESP32 board.

For the Sensor Unit (ESP32-C3):

  1. Open `firmware/Float_Sensor.ino`.
  2. Paste the Receiver's MAC address you copied earlier into the `broadcastAddress[]` array within this file.
  3. Select your ESP32-C3 board from the board manager.
  4. Upload the sketch to your ESP32-C3 board.

3. Final Assembly & Power On

  • Connect the float switches, LEDs, and resistors to the boards as shown in the Schematic diagram.
  • Power on both units. The Receiver will create a Wi-Fi Access Point named "IoT_Flood_Monitor_AP".
  • You can connect to this Wi-Fi network to access the web dashboard and monitor the system.

Usage Guide

Interpreting the Light Signals

  • GREEN LIGHT: Normal traffic flow. The road is clear and safe to pass.
  • PULSING YELLOW LIGHT: Low-level flood detected. Proceed with caution as the road is wet.
  • SOLID RED LIGHT: High-level flood detected. The road is impassable. Do not proceed.
  • BLUE LIGHT: This is a general flood indicator. It will be ON during both Low and High-level flood states.

Accessing the Web Dashboard

  1. On your phone or computer, scan for Wi-Fi networks.
  2. Connect to the network named "IoT_Flood_Monitor_AP".
  3. Use the password: "floodmon" (or the password you configured).
  4. Once connected, open a web browser. You should be automatically redirected to the login page. If not, navigate to the device's IP address (usually 192.168.4.1).
  5. Log in with the configured username and password (default is `admin`/`password`).

Using the Dashboard Controls

The web dashboard allows for manual control and system resets. You can turn individual LEDs on or off for testing, force the system back into its automatic traffic light cycle, or perform a full reset to the NORMAL state.

Future Improvements

Cloud Integration & Mobile Alerts

Connect the Receiver Unit to a home Wi-Fi network to push status updates to an IoT cloud platform. This would enable sending real-time mobile notifications (SMS, Push) to users, ensuring they are alerted to flood conditions even when away from the physical location.

Solar-Powered Sensor Unit

To increase the autonomy and placement flexibility of the remote Sensor Unit, it could be equipped with a small solar panel and a rechargeable battery pack. This would make it ideal for long-term deployment in outdoor or remote locations without access to mains power.

Advanced Sensing & Data Logging

Upgrade the binary float switches to an ultrasonic or ToF (Time-of-Flight) distance sensor. This would provide continuous, granular data on the water level, not just discrete high/low points. This data could be logged to an SD card or the cloud for historical analysis of flood patterns.

Documentation & Process

System Assembly & Receiver Hub

The build process involves wiring the float switches to the Sensor Unit and the LED indicators to the Receiver Unit. Both units are housed in protective casings to shield them from environmental factors. The Receiver Unit is the core of the system, featuring visual alerts and network capabilities. It's configured to run a local Wi-Fi access point, allowing for a direct connection to its web dashboard without needing an external router.

Schematic

Circuit Diagram

The schematic illustrates the connections for both the Sensor and Receiver units. The Sensor Unit uses GPIO pins 4 and 5 for the low and high float switches, configured with internal pull-up resistors. The Receiver Unit uses GPIO pins 18, 21, 22, and 19 to control the red, yellow, green, and blue LEDs, respectively.

Libraries Used

esp_now.h

Utilized for reliable and fast peer-to-peer communication between the ESP32 sensor and receiver units without the overhead of a traditional Wi-Fi connection.

WiFi.h

Enables the ESP32 boards to manage Wi-Fi connections. It is used by the Sensor Unit to connect to the receiver's access point and by the Receiver Unit to create its own Soft AP.

WebServer.h

Used by the Receiver Unit to create a lightweight, secure web server. This server hosts a dashboard for monitoring flood status and provides control endpoints.
